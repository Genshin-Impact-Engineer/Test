# 1 研究目的及意义
为构建一体化本地交互体系，该系统将通过 OLED 屏实时显示商品信息、4路独立按键完成品类选择与参数设置、语音模块实现自动播报，提供优质且丰富的人机交互体验，简化线下结算操作流程。
在通信与数据管理方面，核心控制器通过蓝牙模块与手机APP实现无线通信[8]。该系统使用搭建蓝牙无线数据传输通道，实现商品类别、单价、商品重量、总价、设备状态与微信小程序同步互通，完成线上线下双端显示、数据查询与远程控制。
完成称量物品超过量程与重量严重抖动的相关预警。集成异常报警、状态指示功能，提升系统运行稳定性与使用安全性，打造适配小型生鲜门店、农贸市场、社区零售的轻量化智能结算解决方案。
# 2 系统整体架构
# 2.1 整体设计思路
该系统拟采用STM32F103C8T6、LED、无源蜂鸣器、HX711压力传感器、OLED、独立按键、ASR-PRO语音模块、BLE5.4蓝牙模块、小程序来进行整个系统的状态显示、商品称重、输入输出控制、语音播报、异常报警、线下线下双端显示与控制。
具体实现如下：STM32F103C8T6作为整系统的主控，负责接收、处理、传输系统的各项数据与指令	；HX-711压力传感器借助金属应变片获得电阻的变化通过AD转化电路将数字量转换位模拟量后发送STM32F103C8T6,其接受到数据后为了减小称量物品的瞬时冲击噪声（脉冲干扰）、周边环境的持续随机噪声（高斯噪声）等干扰，采用一阶滞后滤波算法来优化数据，随后将数据进行转化来完成重量的测量；OLED负责显示生鲜的品类、重量、单价、总价；独立按键来作为输入进行选择生鲜的名称、调整生鲜的单价、进行总价的确定；ASR-PRO语音模块来进行生鲜信息的播报；BLE5.4蓝牙模块将各项数据同步上传至小程序实现线上的操控，查阅，LED作为整个系统的心跳灯，来表明系统正常运行与否。

2.2.1 电源系统
为了保障系统能够长期高效的运行，该系统决定使用12V锂电池来进行供电。使用该电源供电主要考虑到其输入输出余量充足，避免电池放电后期电压下降、转换器输出不稳定、系统欠压重启的风险。系统中所用模块对输入电压有着5V、3.3V的要求，对于12V的输入电压使用LM2596S开关电源转化为5V来供电，3.3V的模块输入电压，借助STM32F103C8T6核心板内部的LDO（AMS1117），将 5V 转为 3.3V来实现；通过上述操作来完成整个电源系统的输入，输出电压需求，为所有模块来进行供电。

# 2.2.2 执行器系统
ASR-PRO语音模块、无源蜂鸣器、LED共同组成执行器系统。该子系统负责主系统负责输出反馈的核心部分，分别从语音与光信号两方面向用户传递设备的运行状态和生鲜的称量结果。ASR-PRO语音模块负责着的语音交互功能。当商品的称重完成后，主控芯片会通过串口向语音模块发送指令，模块收到后会自动播报当前商品的名称、重量、预设的单价以及计算得出的总价，提醒服务人员与顾客。另外，当出现称量商品超过HX711量程后，ASR-PRO语音模块会进行报警，提醒服务人员与顾客拿走称量商品，以免长时间放置导致损坏。
无源蜂鸣器内部无驱动电路，需要外部 PWM 信号驱动发声。本系统通过 TIM2_CH1 (PA0) 输出 3kHz 方波（占空比 50%）驱动蜂鸣器，主要负责产生操作提示音与报警。当系统正常运行时，每次按下独立按键，TIM2 输出 100ms 的 3kHz PWM 方波，蜂鸣器发出一声短促提示音；当处于报警页面时，PWM 持续输出，蜂鸣器一直鸣叫；退出报警页面后若报警条件仍在，PWM 以间歇方式（鸣 300ms / 停 300ms）输出，提醒工作人员进行处理。
板载LED是整个系统的心跳灯，负责反应整个软件系统的运行状态。该LED灯连接在STM32F103C8T6的PC13引脚，通过GPIO输入输出引脚控制，低电平点亮，高电平熄灭。当系统出现程序卡死、死循环或者逻辑错误时，心跳等出现常亮或者常灭等状态；当出现其他问题例如通信超时，系统等待时，心跳灯的反转频率会发生改变，来帮助用户判断故障类型。除此之外，信号灯的供电由核心板内部LDO的输出的3.3V供电，因此哪怕其他的执行器出现故障，只要主控一直在运行，心跳灯也可以做到最基本的状态指示。执行器系统如下图2-3所示。

# 2.2.3 输入系统
输入系统由独立按键和HX-711压力传感器模块两部分构成。
独立按键一共有四个(KEY1、KEY2、KEY3、KEY4)，用于进行称重相关的操作。例如选择生鲜的品类、调整单价、取消调整、确认称重等功能，按键直接连接主控对应GPIO，通过电平的检测实现快速响应，GPIO引脚设置为上拉输入，当按下按键时，引脚电平被拉低，确认按下按键，主控执行相关操作。
HX-711模块连接应变式称重传感器，通过金属应变片电阻的变化来将将物品重量转换为高精度数字信号，并通过专用串行接口来进行数据的传输。STM32最小系统板为了接收发送的数据，需要为HX-711模块编写相应的驱动。通过一个专用的，类似SPI的两行串口来进行连接，不过在硬件上并非标准的IIC或者SPI接口，因此必须连接到主控的GPIO口，通过编写的驱动来模拟时序然后接受到对应的数据。
自定义同步串口由SCK (时钟和数据输入)与DOUT (数据输出)组成。
SCK ：由STM32最小系统板控制，通过向HX711发送时钟脉冲，用来读取数据并选择下一次的转换参数。DOUT ：HX711通过此引脚向STM32最小系统板发送转换完成的24位数据。当STM32最小系统板完成数据的接受后，会利用一阶滞后滤波算法，来减小称量物品的瞬时冲击噪声（脉冲干扰）、周边环境的持续随机噪声（高斯噪声）等干扰，在低延时与低数据抖动之间取得平衡。输入系统如图2-4所示。

# 2.2.4 数传数显系统
数据的显示与传输时本系统需要重点解决的问题之一。为了方便数据的显示与查阅，本系统采用OLED屏幕与微信小程序实现线上线下同步显示，通过BLE5.4蓝牙模块实现STM32F103C8T6最小系统板和小程序之间的通信。
当STM32F103C8T6接收到HX-711压力传感器返回的数据后，为了减少数据在OLED屏幕上的抖动次数，采用一阶滞后滤波算法减少数据的抖动，提升商品的结算效率，处理后的数据将通过OLED屏幕来进行显示。
OLED屏幕选用4线IIC通讯版本(VCC、GND、SCL、SDA）来进行显示。该版本的OLED的接线简单，有着统一标准，驱动的编写方便易上手，通讯速率满足整个系统的需求，适合项目静态文字，简单图形且低刷新率的显示方案。无线通信技术，凭借其快捷的传输速度、便利的接入方式、高度的可移动性以及强大的安全性能，被广泛应用在了各行各业之中，成为推动社会现代化进程的重要力量[22]。
数据的传输使用BLE5.4蓝牙模块通过UART串口通信协议，该协议是嵌入式开发中常用的通信协议，对电路的要求非常简单，通信协议也比较简洁，便于开发。全双工的通信需求十分满足生鲜电子秤结算系统的数据上传与指令下发，多种数据帧与通信速率（波特率）可供选择，本项目采用UART通信协议中常用的数据帧格式：8位数据位、无校验位、1位停止位。拟采用BLE5.4蓝牙模块可供支持的最大115200的通信速率实现数据快速收发，为便于数据实时查看。数据显示与传输系统技术路线图如图2-4所示。

# 4.1 系统时钟与定时器资源配置

STM32F103C8T6 共计 4 个硬件定时器（TIM1~TIM4）加上系统 SysTick。本系统仅需使用 **2 个定时器**：SysTick + TIM2，剩余 TIM1/3/4 预留扩展。

**资源分配如下：**

| 定时器 | 用途 | 关键配置 |
|--------|------|----------|
| **SysTick** | HAL_Delay 延时基准 | 1ms 中断，CubeMX 自动配置 |
| **TIM2** | 统一时基，驱动所有周期任务 | 1ms 中断，预分频 72-1，周期 1000-1 |
| TIM1/3/4 | 预留 | 不使用 |

**TIM2 中断服务函数中维护软件计数器，派生以下周期任务：**

| 任务 | 周期 | 说明 |
|------|------|------|
| 按键扫描 + 消抖 | 20ms | 轮询 4 路 GPIO，软件去抖 |
| HX711 数据采集触发 | 100ms（10Hz） | 定时发起采样，保证采样率稳定 |
| LED 心跳灯翻转 | 正常 500ms；异常可变 | 1Hz 闪烁表明系统正常 |
| 蜂鸣器短鸣关闭 | 持续 50ms 后关断 | 按键提示音 |
| 蜂鸣器报警循环 | 鸣 300ms / 停 300ms | 异常报警间歇鸣叫 |
| 重量稳定超时计时 | 5s | 物品放置后判断是否稳定 |

TIM2 作为 1ms 统一时基，所有周期任务通过软件计数器分频产生，避免为每个任务单独占用硬件定时器。该方案仅占用 1 路硬件定时器资源即可覆盖全部周期性驱动需求。

# 4.3 数据采集程序设计

HX711采样数据到达STM32后，采用**一阶滞后滤波**（一阶低通滤波）进行处理，替代常见的滑动平均+中值滤波的双算法级联方案，以更低的运算开销和更低的数据延时满足系统需求。

**滤波公式：**

$$Y(n) = \alpha \cdot X(n) + (1-\alpha) \cdot Y(n-1)$$

其中 $X(n)$ 为当前采样值，$Y(n-1)$ 为上一次滤波输出值，$Y(n)$ 为本次滤波输出值，$\alpha$ 为滤波系数（0 < $\alpha$ < 1）。

**参数选取：**

$\alpha$ 取 **0.3**。此值下滤波输出可在约 10 个采样周期内收敛至真实值的 95% 以上，对冲击噪声有足够的平滑能力，同时响应迅速，满足称重实时显示需求。若现场环境噪声偏大，可将 $\alpha$ 下调至 0.15~0.2 以增强平滑效果。

**算法特点：**

- 计算量极小：仅需 1 次乘法 + 1 次加法，适配 STM32F103C8T6 的算力
- RAM 占用 1 个变量：仅保存上一轮滤波输出值
- 延时可控：通过调整 $\alpha$ 在 "响应速度" 与 "数据平滑度" 之间灵活平衡
- 对脉冲干扰（物品放置瞬间的冲击）和高斯噪声（环境随机抖动）均有抑制作用

滤波处理后的重量数据送入 OLED 显示模块和蓝牙传输模块。重量稳定判定条件：连续 10 次滤波输出值的波动范围 < 5g，认为称重已稳定。

# 4.5 语音播报与蜂鸣器提示程序设计
该系统中语音模块的播报与蜂鸣器产生的指示音/报警音是操作者在进行获取称量时的一个重要的反馈。
语音播报由ASR-PRO模块完成，有STM32F103C8T6最小系统板通过串口发送指令触发。串口采用波特率采用115200，数据帧采用常见的8N1格式：8位数据位、无校验位、1位停止位、每字节共10位。本系统中语音模块ASR-PRO与STM32F103C8T6最小系统板通信的数据线较短，且环境中的干扰较少，因为上述数据帧刚好能满足与语音模块通信的需求。
语音播报的内容分为操作确认类故障提示两大类。操作确认播报（当操作者在称量界面或调整界面由通过指定按键操作时触发）：当称量界面未选中光标时，按下KEY3，语音模块播报”商品结算与该商品价格”。当称量界面未选中光标时，按下KEY4，商品重新称量，语音模块播报该商品的类别、单价、质量、总价。当调整界面按下KEY3确认修改后，语音模块播报”调整完成”。当调整界面按下KEY4，取消修改后，语音模块播报”取消调整”。故障提示播报（当系统处于异常状态时自动触发）：当称重商品重量超过预设量程时，语音模块播报“商品过重”。当物品放置后5秒内重量仍未稳定（严重抖动）时，播报“重量异常”。上述所有语音播报在单次使用时只执行一次，避免重复干扰。
为了提高代码的可读性和后续的迭代，系统中所用的蜂鸣器采用无源蜂鸣器，通过 TIM2_CH1 (PA0) 输出 3kHz PWM 方波驱动其鸣叫，占空比 50%。
蜂鸣器具有于短促提示音和持续报警音两种声音功能。按键提示音：在三个界面（称量界面、调整界面、故障报警界面）时，每次有效按键（KEY1～KEY4）按下时，蜂鸣器发出约50ms的“嘀”声，表示按键已经被识别。该提示音不区分界面和光标状态，用来提供统一操作反馈。异常报警音：当系统进入故障报警界面时（商品自重超过电子秤量程或者商品重量长时间不稳定），蜂鸣器将会以间歇方式持续鸣叫（鸣300ms、停300ms循环），直至操作者按下任意键退出报警界面，蜂鸣器停止报警。为了防止误触，按键提示音需要配合软件消抖，避免因触点抖动产生多次短鸣，影响操作。
程序逻辑中要设置异常报警音的优先级高于按键提示音；当报警发生时，若同时有按键操作，仍先响应报警逻辑。为了保障上述功能的独立性，语音播报与无源蜂鸣器控制分别由独立的GPIO和串口指令实现，互不干扰，避免一方出问题，语音报警功能或按键提示功能瘫痪。通过上述设计，系统能够为操作者提供清晰、及时的听觉反馈，有效提升称重终端的易用性和安全性。

# 4.6 数据上传程序设计
系统采用小机云BLE5.4蓝牙模块作为通信中间件，实现 STM32F103C8T6最小系统板与开发商免费提供的微信小程序之间的远程数据交互。BLE5.4蓝牙模块与 STM32 通过UART 串口进行全双工异步通信，通信波特率配置为115200 bps，数据位 8 位，停止位 1 位，无校验位，无硬件流控，保证指令与数据传输的稳定可靠。该系统的数据传输通过串口的稳定传输以及蓝牙模块优秀的抗干扰功能设计能够在商城等复杂环境高效保证数据的传输，保障系统对远程实时监测与智能控制的需求。
为将测量的商品重量、单价、质量、以及总价通过串口发送给BLE5.4蓝牙模块，随后再将数据发送给微信小程序，通过对应的微信小程序查阅数据、下发指令，实现数据的双端传输。
