# OLED 显示模块指南（电子秤/POS 系统）

## 一、硬件规格

| 参数 | 规格 |
|------|------|
| 屏幕类型 | SSD1306 OLED |
| 分辨率 | 128 × 64 像素 |
| 接口 | I2C（I2C1, SCL=PB8, SDA=PB9）|
| 从机地址 | 0x78（7-bit） |
| 字体 | 8×16 像素 ASCII，4 行 × 16 列 |
| 帧缓冲 | 1024 字节（128×64 / 8） |

## 二、界面总览

三页面系统，循环切换：

```
称重页 (PAGE_WEIGHING)  ←→  调整界面 (PAGE_PREVIEW)
        ↑                           │
        └──────── 报警页（报警才触发） ───────────┘
              (PAGE_ALARM)
```

**注意**：页面切换由按键逻辑控制：
- 称重 → 调整：在对应属性栏，按下KEY2 进入
- 调整 → 称重：KEY2 保存或 KEY3 取消
- 报警触发：重量异常自动跳入
- 报警退出：任意按键返回称重页

## 三、Page_one：称重页

### 布局

```
┌────────────────────────────────┐
│ 商品:苹果                      │  ← 第 0 行（可选中，item=1）
│ 单价:8.00元/kg                 │  ← 第 1 行（可选中，item=2）
│ 重量:1.25kg                    │  ← 第 2 行（只读，实时刷新）
│ 总价:10.00元                   │  ← 第 3 行（只读，自动计算）
└────────────────────────────────┘
```
去皮激活时，第 2 行尾部显示去皮值（正数、2 位小数、300ms 闪烁）：
```
│ 重量:0.50kg 0.30               │  ← 净重 + 皮重（闪烁）
```
重量/单价/总价/去皮值均为 2 位小数。

### 可操作项

| 操作 | 按键 | 条件 | 效果 |
进入称重界面，默认光标未选择（0）
|------|------|------|------|
| 光标上移 | KEY4 | 未编辑 | 0 → 单价 → 类别 → 0（循环）|
| 光标下移 | KEY3 | 未编辑 | 0 → 类别 → 单价→ 0（循环）|

| 进入编辑 | KEY2 | 选中类别/单价 | 进入编辑模式（闪烁） |
| 去皮/取消 | KEY1 | 未选中 | 按下→记录当前毛重为皮重，净重归零，去皮值闪烁；再次按下→取消去皮 |
| 退出选中 | KEY1 | 选中类别/单价 | 取消选中 |
| 退出编辑 | KEY1 | 编辑模式 | 退出编辑 |

| 上一商品 | KEY4 | 编辑类别 | 切换到下一个商品（单价及对应总价自动更新） |
| 下一商品 | KEY3 | 编辑类别 | 切换到下一个商品（单价及对应总价自动更新） |
| 单价+0.5 | KEY4 | 编辑单价 | 商品单价加 0.5，上限 200 |
| 单价-0.5 | KEY3 | 编辑单价 | 商品单价减 0.5，下限 0 |

| 确认 | KEY2 | 编辑模式 | 保存数据，退出编辑 |（如果更新的是单价，则库中商品对应的单价进行更新）

## 五、Page_three：报警页

### 布局（超重）

```
┌────────────────────────────────┐
│ !!   WARNING   !!              │  ← 第 0 行
│ Overweight!                    │  ← 第 1 行
│ Remove items                   │  ← 第 2 行
│ Press any key...               │  ← 第 3 行
└────────────────────────────────┘
```
/*超重值预设为2kg/
### 布局（重量异常）

```
┌────────────────────────────────┐
│ !!   WARNING   !!              │
│ Weight Error!                  │
│ Please re-weigh                │
│ Press any key...               │
└────────────────────────────────┘
```

### 触发条件

| 类型 | 条件 | 阈值 |
|------|------|------|
| 超重 | 重量 > 2.0 kg | WEIGHT_MAX |
| 重量异常 50g上下跳动超过3s  |

### 退出方式

**任意按键**（KEY0/KEY1/KEY2/KEY3）退出报警页面，返回称重页
注意：退出报警界面不代表退出报警状态，如果退出报警界面，可是依旧超重，或者重量异常，依旧是报警状态
## 六、代码架构

```
oled.h / oled.c
├── OLED_Setup()             → OLED 初始化 + 默认参数
├── Boot_Interface_Show()    → 开机动画（进度条）
├── OLED_UpdateDisplay()     → 主刷新循环（渲染 + I2C 故障恢复）
├── OLED_HandleKey()         → 三页面按键分发
├── OLED_SetAlarmFlags()     → 设置报警标志
├── OLED_SetCategoryByName() → 蓝牙命令：设置类别
├── OLED_SetUnitPrice()      → 蓝牙命令：设置单价
└── Scale_TriggerReweigh()   → 触发重称
```

## 七、I2C 故障恢复

STM32F1 I2C 外设存在 BUSY 锁死缺陷。本模块内置双重恢复机制：

1. **每次刷新前主动探测**: 发送 `ENTIRE_DISPLAY_OFF` 命令，超时 5ms
2. **定期强制恢复**: 每 200 次刷新（约 100 秒）执行一次 I2C 外设复位
3. **Flush 失败即时恢复**: `OLED_Flush()` 返回非 OK 时立即恢复并重试

恢复步骤：
```
FORCE_RESET I2C1 → 延时 1ms → RELEASE_RESET → MX_I2C1_Init() → OLED_Init()
```

## 八、视觉效果

| 效果 | 说明 |
|------|------|
| 选中框 | 128×16 空心矩形框包围当前选中行 |
| 闪烁 | 编辑模式下，选中行每 300ms 切换显示/空白 |
| 刷新率 | 正常模式 500ms，编辑模式 300ms（闪烁同步） |

## 九、商品表

商品表定义在 `product_table[PRODUCT_COUNT]`，每个商品包含名称和默认单价：

```c
const Product_t product_table[PRODUCT_COUNT] = {
    {"Apple",  5.0f},
    {"Banana", 3.0f},
    // ... 最多 8 种商品
};
```
/*
苹果，8.0
香蕉，7.0
菠萝，8.0
葡萄, 6.0
西瓜, 4.0
梨子，12.0
哈密瓜：12*/
**扩展方法**：修改 `PRODUCT_COUNT` 宏并添加表项即可。
