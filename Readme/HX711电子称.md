1. 模块本质与用途
HX711 是一款专用高精度 24 位 A/D 转换器，仅用于电阻应变式称重 / 压力传感器的微弱差分信号采集。
它不具备通用 ADC 功能，专为电子秤、测力设备设计。
2. 核心工作原理（严格版）
2.1 信号输入原理
模块采用差分输入，只识别 INA+ 与 INA- 之间的电压差。
传感器受力 → 电桥失衡 → 输出 μV~mV 级微弱电压。
该信号无法被单片机直接识别，必须进入 HX711 放大、转换。
2.2 内部处理流程（固定不可修改）
差分输入 → 低噪声放大器（PGA）
默认增益 128 倍，将小信号放大到适合 AD 转换的范围。
放大信号 → 24 位 Σ-Δ 型 AD 转换
将连续模拟信号转为 24 位数字量。
数字信号 → 内部时钟同步输出
输出速率固定为 10Hz 或 80Hz（由 RATE 引脚决定）。
数据就绪 → DOUT 拉低
通知单片机可以读取。
2.3 通信原理（严格时序）
HX711 与 MCU 采用 同步二线通信：
PD_SCK：单片机输出时钟，控制读取与增益配置。
DOUT：HX711 输出数据，低电平表示数据就绪。
严格时序规则
上电后，PD_SCK 必须保持低电平，模块才能正常转换。
DOUT 为高 → AD 正在转换，不可读取。
DOUT 为低 → 转换完成，可以读取。
单片机发送 24 个脉冲 → 读出 24 位数据。
继续发送 第 25 个脉冲 → 设置下一次为 A 通道、增益 128。
读取完成后，PD_SCK 必须回到低电平，否则模块进入断电状态。
2.4 通道与增益原理（严格固定）
由 PD_SCK 脉冲总数 决定下一次转换模式：
25 个脉冲 → A 通道 / 增益 128（电子秤标准配置）
26 个脉冲 → B 通道 / 增益 32
27 个脉冲 → A 通道 / 增益 64
⚠️ 脉冲数 不能少于 25，不能多于 27，否则通信失效。
2.5 数据格式原理
输出为 24 位二进制补码：
最高位为符号位
无负载时理论值为 0x800000
通常软件处理：Data = Raw ^ 0x800000 转为无符号值
3. 电气规范（必须遵守）
3.1 供电要求
供电电压：4.7V ~ 5.5V
低于 4.7V：数据漂移、不准、无法工作
高于 5.5V：芯片永久损坏
3.2 输入信号要求
差分输入电压：±20mV（增益 128 时）
超出范围 → 数据饱和、不再随重量增加
3.3 空闲通道处理
B 通道（INB+、INB-）不用时必须接地，否则引入干扰导致数据跳动。
4. 严格使用步骤（工程版）
4.1 硬件接线步骤
传感器四线接入：
激励 + → E+
激励 - → E-
信号 + → A+
信号 - → A-
HX711 与 MCU 接线：
PD_SCK → 推挽输出 GPIO
DOUT → 上拉输入 GPIO
所有地共地，避免电势差。
4.2 初始化步骤
驱动 PD_SCK 为低电平。
等待 ≥400ms 让 AD 稳定。
连续读取若干次丢弃，确保进入稳定状态。
4.3 读取步骤（严格）
等待 DOUT = 低。
输出 24 个时钟，读取 24 位数据。
输出第 25 个时钟，设置下一次 A 通道 / 128 增益。
拉低 PD_SCK，结束时序。
做符号位处理：data ^= 0x800000。
4.4 校准与使用
空载校准：读取并保存基准值。
砝码标定：用已知重量计算系数。
实时重量公式
重量 = (当前AD值 - 空载AD值) × 标定系数
5. 严格禁止行为
禁止 PD_SCK 持续高电平超过 60μs → 模块直接断电。
禁止读取时脉冲数 <25 或>27 → 数据错误、模块锁死。
禁止模拟地与数字地分离布线 → 干扰巨大。
禁止在电源不稳定时使用 → 数据漂移不可修复。
禁止将模块用于非应变式传感器 → 不匹配、损坏风险。
6. 正常工作判断标准
空载时数据在小范围内波动。
加压时数值单调上升。
去掉压力后回到原点。
10Hz 模式下每秒稳定输出 10 次数据。
流程图
上电初始化
    |
    ↓
PD_SCK置低 → 模块开始转换
    |
    ↓
等待DOUT变低（数据就绪）
    |
    ↓
24个时钟 → 读取24位原始数据
    |
    ↓
第25个时钟 → 设置下次A通道/128增益
    |
    ↓
PD_SCK置低 → 本次读取完成
    |
    ↓
数据校正 → 滤波 → 重量计算

| PD_SCK | PA6 | 推挽输出 (OUTPUT_PP) | MCU 输出时钟 |

```c
/* PA6: HX711 SCK —— 必须初始化为低电平 */
HAL_GPIO_WritePin(GPIOA, GPIO_PIN_6, GPIO_PIN_RESET);

/* PA7: HX711 DOUT —— 上拉输入 */
GPIO_InitStruct.Pin = GPIO_PIN_7;

HAL_GPIO_Init(GPIOA, &GPIO_InitStruct);
```

## 8. 微秒延时

8MHz HSI 下，约 8 次空循环 ≈ 1μs。用于 HX711 时序中的脉冲间隔：

```c
#define HX711_US_DELAY()  do { volatile int _d = 8; while (--_d); } while(0)
```

## 9. 底层读取函数 `HX711_Read()`

严格遵循 HX711 时序：等待 DOUT 变低 → 发 24+1 个脉冲读取数据 → 异或 0x800000 转无符号值。

```c
static int32_t HX711_Read(void)
{
    int32_t data = 0;
    uint32_t timeout = 200000;  /* 超时保护 */

    /* 1. 等待 DOUT 变低（数据就绪） */
    while (HAL_GPIO_ReadPin(HX711_DOUT_PORT, HX711_DOUT_PIN) == GPIO_PIN_SET) {
        if (--timeout == 0) return -1;  /* 超时 → 返回 -1 */
    }

    /* 2. 发送 25 个时钟脉冲 */
    for (int i = 0; i < 25; i++) {
        HAL_GPIO_WritePin(HX711_SCK_PORT, HX711_SCK_PIN, GPIO_PIN_SET);
        HX711_US_DELAY();
        if (i < 24) {
            /* 前 24 个脉冲：在时钟高电平期间读取 DOUT */
            data = (data << 1) | HAL_GPIO_ReadPin(HX711_DOUT_PORT, HX711_DOUT_PIN);
        }
        /* 第 25 个脉冲：设置下一次为 A 通道 / 增益 128 */
        HX711_US_DELAY();
        HAL_GPIO_WritePin(HX711_SCK_PORT, HX711_SCK_PIN, GPIO_PIN_RESET);
        HX711_US_DELAY();
    }

    /* 3. 补码转无符号 */
    data ^= 0x800000;
    return data;
}
```

**关键细节**：
- 超时值 200000 对应约 200ms（8MHz 下），防止传感器断连时死循环
- 第 25 个脉冲只配置通道/增益，不读数据
- `^= 0x800000`：24 位二进制补码 → 无符号偏移量

## 10. 开机自动调零 `Sensor_Init()`

### 流程

```
PD_SCK 置低 → 等待 500ms（模块稳定）
    ↓
丢弃前 5 次读数（上电不稳定）
    ↓
采集 20 次 → 冒泡排序 → 去首尾各 3 个 → 取均值 → 作为零位偏移
    ↓
初始化滤波器状态和滑动缓冲
```

### 代码

```c
void Sensor_Init(void)
{
    /* 1. 确保 SCK 低电平，等待模块稳定 */
    HAL_GPIO_WritePin(HX711_SCK_PORT, HX711_SCK_PIN, GPIO_PIN_RESET);
    HAL_Delay(500);

    /* 2. 丢弃前 5 次不稳定读数 */
    for (int i = 0; i < 5; i++) {
        HX711_Read();
    }

    /* 3. 开机自动调零: 20 次采样 → 排序 → 去首尾各 3 → 均值 */
    #define TARE_SAMPLES 20
    #define TRIM_EDGE     3
    int32_t s[TARE_SAMPLES];
    int n = 0;
    for (int i = 0; i < TARE_SAMPLES; i++) {
        int32_t raw = HX711_Read();
        if (raw >= 0) s[n++] = raw;
    }

    /* 冒泡排序 */
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - 1 - i; j++) {
            if (s[j] > s[j + 1]) {
                int32_t t = s[j];
                s[j] = s[j + 1];
                s[j + 1] = t;
            }
        }
    }

    /* 去掉最小 3 个和最大 3 个，取均值 */
    int trim = (n > TRIM_EDGE * 2 + 2) ? TRIM_EDGE : 0;
    int64_t sum = 0;
    for (int i = trim; i < n - trim; i++) sum += s[i];
    zero_offset = (int32_t)(sum / (n - 2 * trim));

    /* 4. 初始化滤波器状态 */
    filtered_w = 0.0f;
    last_out   = 0.0f;
    buf_idx    = 0;
    buf_full   = 0;
    for (int i = 0; i < BUF_SZ; i++) raw_buf[i] = 0;
}
```

**为什么去首尾**：消除偶然的脉冲干扰（如开机瞬间的电压波动），提高零位精度。

## 11. 滑动均值降噪 `moving_avg()`

在原始 AD 值进入滤波器之前，先做 4 样本滑动均值，削弱单次读数的随机抖动：

```c
#define BUF_SZ 4
static int32_t raw_buf[BUF_SZ];
static uint8_t buf_idx = 0;
static uint8_t buf_full = 0;

static int32_t moving_avg(int32_t new_val)
{
    raw_buf[buf_idx] = new_val;
    buf_idx = (buf_idx + 1) % BUF_SZ;
    if (buf_idx == 0) buf_full = 1;  /* 缓冲区满后始终取 4 样本 */

    int32_t sum = 0;
    int n = buf_full ? BUF_SZ : buf_idx;
    for (int i = 0; i < n; i++) sum += raw_buf[i];
    return sum / n;
}
```

## 12. 自适应一阶滞后滤波 `adaptive_filter()`

### 算法公式

$$Y(n) = \alpha \cdot X(n) + (1-\alpha) \cdot Y(n-1)$$

### 自适应策略

| 条件 | α 值 | 效果 |
|------|------|------|
| 变化量 > 10g | 0.5（快速） | 快速跟踪重量变化 |
| 变化量 ≤ 10g | 0.1（慢速） | 强力平滑稳态噪声 |

### 代码

```c
#define FILTER_ALPHA_FAST   0.5f
#define FILTER_ALPHA_SLOW   0.1f

static float adaptive_filter(float raw_w)
{
    float diff = raw_w - filtered_w;
    if (diff < 0) diff = -diff;

    float alpha = (diff > 0.010f) ? FILTER_ALPHA_FAST : FILTER_ALPHA_SLOW;
    filtered_w = alpha * raw_w + (1.0f - alpha) * filtered_w;
    return filtered_w;
}
```

**设计考量**：
- 物品放上瞬间（变化大）→ α=0.5，约 4 个采样周期收敛至 90%
- 稳态称重（变化小）→ α=0.1，强力抑制环境随机噪声
- 仅需 1 次乘法 + 1 次加法，RAM 占用 1 个 float（4 字节）

## 13. 输出死区 `OUTPUT_DEADBAND`

滤波后的重量变化 < 2g 时不更新输出，抑制尾数跳变：

```c
#define OUTPUT_DEADBAND     0.002f  /* 2g */

/* 在 Sensor_Update() 中 */
float w = adaptive_filter(raw_w);
float diff = w - last_out;
if (diff < 0) diff = -diff;
if (diff >= OUTPUT_DEADBAND) {
    last_out = w;
}
```

## 14. 完整采样流程 `Sensor_Update()`

```c
void Sensor_Update(void)
{
    /* Step 1: 读原始 AD 值 */
    int32_t raw = HX711_Read();
    if (raw < 0) return;  /* 超时 → 跳过本次 */

    /* Step 2: 滑动均值降噪 */
    raw = moving_avg(raw);

    /* Step 3: 去零位偏移 → 乘标定系数 → 得重量(kg) */
    float raw_w = (float)(raw - zero_offset) * scale_factor;

    /* Step 4: 一阶自适应滤波 → 死区判断 → 更新输出 */
    float w = adaptive_filter(raw_w);
    float diff = w - last_out;
    if (diff < 0) diff = -diff;
    if (diff >= OUTPUT_DEADBAND) {
        last_out = w;
    }
}

float Sensor_GetWeight(void)
{
    return last_out;
}
```

**信号处理链路**：
```
AD原始值 → 滑动均值(4样本) → 去零位×系数 → 自适应一阶滤波 → 死区 → 输出重量
```

## 15. 标定系数

```c
#define DEFAULT_SCALE_FACTOR   0.00000267f
```

**标定方法**：
1. 空载开机，自动调零（zero_offset）
2. 放置已知重量砝码（如 200g = 0.200kg）
3. 读取当前 AD 值（已减去 zero_offset）
4. 计算系数：`scale_factor = 0.200 / (AD值 × 当前系数初值)`
5. 反复验证：用不同砝码（15g、50g、200g）交叉校验

**实测数据**（供参考）：
- 15g 砝码 → 显示 0.014kg（误差 1g）
- 0.213kg 物品 → 显示 0.188kg（连续修正后）

## 16. 主循环调用方式

`Sensor_Update()` 由 100ms 周期任务驱动，保证 10Hz 采样率（与 HX711 默认输出速率匹配）：

```c
/* 主循环 main.c */
while (1) {
    uint32_t now = sys_tick_ms;

    /* 100ms 周期任务 */
    if (task_flag_100ms) {
        task_flag_100ms = 0;
        Sensor_Update();
        float w = Sensor_GetWeight();
        /* w 即为当前重量(kg)，送入 OLED 显示和报警检测 */
    }
}
```

## 18. 故障排查

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| 重量始终为 0 | HX711 断连 / SCK 未初始化 | 检查 PA6、PA7 波形 |
| 重量剧烈跳动 | 传感器 B 通道未接地 | INB+、INB- 短接到 GND |
| 重量漂移 | 供电不稳 / 标定系数不准 | 用万用表测 5V，重新标定 |
| 开机后偶尔卡死 | DOUT 一直为高（HX711 未启动） | 检查 PD_SCK 开机是否为低 |
| 读数超时（返回 -1） | DOUT 引脚断开或传感器断电 | 检查接线 + 传感器是否供电 |
