# 小机云蓝牙开发指南（STM32 + X-B01）

## 一、硬件连接

### 1. X-B01 引脚定义

| 引脚 | 名称 | STM32 连接 |
|------|------|-----------|
| VCC | 电源正极 | 3.3V |
| GND | 地 | GND |
| TXD | 发送端 | USART_RX (PA3) |
| RXD | 接收端 | USART_TX (PA2) |
| GPIO | 模式选择 | LOW=透传，HIGH=AT 命令 |

### 2. STM32F103C8T6 串口配置

```c
// USART1: PA9(TX) - PA10(RX)
// USART2: PA2(TX) - PA3(RX) → 连接蓝牙
void UART_Init(void) {
    // 波特率：115200
    // 数据位：8, 停止位：1, 校验：无
}
```


## 二、双向通信流程

### 完整数据流

```
┌─────────────┐     JSON 字符串      ┌──────────┐
│   STM32     │ ─────────────────→ │  云端    │
│             │ ← ──────────────── │  平台    │
│ UART 发送   │      AT 指令       │          │
└─────────────┘                    └────┬─────┘
                                        │
                                        ▼
                                  ┌─────────────┐
                                  │   小程序    │
                                  │             │
                                  │ 蓝牙接收    │
                                  └─────────────┘
```

### 时序说明

1. **STM32 → 云端**：定时上报传感器数据（如每 0.5 一次）
2. **云端 → STM32**：小程序操作后下发控制指令
//3. **STM32 响应**：执行动作后再次上报状态确认

文本协议
# 协议格式
$X#Cmd#Content&CheckCode\r\n
$ 符号：协议开始符号
& 符号：内容分割符
符号：协议结束符，十六进制0x0A 0x0D
# 符号：内容分割符，对协议内容进行分段。每段内容描述如下：
$X：协议固定开头
Cmd：协议功能，如Cmd为DATA为上传数据。部分指令功能码有简写，如DATA，简写为D。可减小发送数据量
Content：协议数据，数据格式见协议详情，注意数据中不可以出现#号。如果没有内容可以不需要该字段。
CheckCode：校验码，&前所有字符进行异或得到 1 个字节的十六进制数，以2 个ASCII 字符形式输出。如以下协议：$X#Cmd#Timestamp#Content&CheckCode\r\n &之前部分异或得到校验码为0x12（BCC校验(异或校验)），则CheckCode位填12
对于设备发给手机的长指令，可以分成几次发送，以\r\n结束


# 协议回复格式
$XA#Cmd#Content&CheckCode\r\n
如果无特殊说明，指令统一回复：
成功：$XA#Cmd#OK&CheckCode
失败：$XA#Cmd#ERR:message&CheckCode
常见的错误回复
ERR:CRC：未进行KEY_CONTENT校验，详见本文发送校验内容一节
ERR:XOR：指令校验码错误
ERR:D：数据格式错误
LOAD_FAIL：加载错误

# 发送数据
CMD：DATA，简写D
格式：$X#D#key1:data1;key2:data2&#CheckCode#\r\n
可以设置一个数组也可以同时设置多组数据，bool内容支持0和1表示
示例1：$X#D#switch:1&21
示例2：$X#D#value:32;enabled1:false;enabled2:0;items:a,b,a;numbers:1,2,3;color:ff0000&1E
回复：$XA#D#OK&7D
给你举个例子
组件		     	 组件属性       绑定字段
浊度  	    	 text          text_turb
温度		   	   text          text_temp
预设温度        number        number_preset_temp
阀门档位        number        number_water_level
温度上限        number        number_temp_max
温度下限        number        number_temp_min
温度阈值        number        number_turb_threshold
温度状态        text          text_temp_state
水质状态        text          text_turb_state
阀门模式        selected      selected_valve_mode
如发送给
测得温度11发送，
如组件表所示，其对应绑定字段为：text_temp
依据协议格式知，应发送：
$X#D#text_temp:11&4C\n
CheckCode根据BBC校验所得，为4C
发送正确，STM32接收到
$XA#D#OK&7d

如果在小程序上控制温度上限，设置值为18.5，
STM32接收到数据：
$X#D#number_temp_max:18.5&6b\n

若利用按键控制温度上限通过单片机发送数据给小程序:
$X#D#number_temp_max:18.5&6b\n
小程序正确接受回单片机：
$XA#D#OK&7d

通过按键，控制单片机发送信息控制阀门模式为自动模式，
小程序正确接受回单片机：
$XA#D#OK&7d

如果在小程序上控制控制阀门模式为自动模式，
STM32接收到数据：
$X#D#selected_valve_mode:0&50\n


所有发送给组件的数据都需要根据协议来写
数据实时传输给小程序，完成相应数据的调整
也实时接受来自小程序的指令，完成相应数据的调整
阈值报警也要快速上传数据至小程序
## 七、关键注意事项

| 问题 | 解决方案 |
|------|----------|
| 蓝牙模块功耗 | 休眠时降低上报频率 |
| JSON 字符串长度 | 控制在 128 字节以内 |
| 浮点数精度 | 使用 `%.1f` 格式化 |
| 帧定界符 | 每条消息以 `\r\n` 结尾 |
| 并发问题 | 使用环形缓冲区处理串口数据 |
| 断线重连 | 实现看门狗自动重启蓝牙连接 |
