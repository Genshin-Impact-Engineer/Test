# 小机云蓝牙开发指南（STM32 + X-B01）

## 一、硬件连接

### 1. X-B01 引脚定义

| 引脚 | 名称 | STM32 连接 |
|------|------|-----------|
| VCC | 电源正极 | 3.3V |
| GND | 地 | GND |
| TXD | 发送端 | USART_RX (PA3) |
| RXD | 接收端 | USART_TX (PA2) |
| GPIO | 模式选择 | LOW=透传，HIGH=AT 命令 |

### 2. STM32F103C8T6 串口配置

```c
// USART1: PA9(TX) - PA10(RX)
// USART2: PA2(TX) - PA3(RX) → 连接蓝牙
void UART_Init(void) {
    // 波特率：9600/115200
    // 数据位：8, 停止位：1, 校验：无
}
```

### 1. 设备上报数据（STM32 → 云端）

#### JSON 字符串格式

```c
char json_buf[64];
float temp = 3.3;
snprintf(json_buf, sizeof(json_buf), "{\"dp\":{\"shsihitemp\":%.2f}}", temp);
UART_Send(json_buf);  // 发送带\r\n 的字符串
```

**发送内容示例**：
```
{"dp":{"shsihitemp":3.30}}\r\n
```

#### AT 指令格式（备用）

```
GPRS_DP_SET shsihitemp=3.30
```

### 2. 云端下发控制指令（小程序 → STM32）

收到指令格式：
```
{"cmd":"set","dp":{"valve_control":true}}
```

**解析后执行**：
```c
if (valve_control == true) {
    GPIO_SetBits(GPIOA, GPIO_Pin_0);  // 打开阀门
} else {
    GPIO_ResetBits(GPIOA, GPIO_Pin_0); // 关闭阀门
}
```

## 五、双向通信流程

### 完整数据流

```
┌─────────────┐     JSON 字符串      ┌──────────┐
│   STM32     │ ─────────────────→ │  云端    │
│             │ ← ──────────────── │  平台    │
│ UART 发送   │      AT 指令       │          │
└─────────────┘                    └────┬─────┘
                                        │
                                        ▼
                                  ┌─────────────┐
                                  │   小程序    │
                                  │             │
                                  │ 蓝牙接收    │
                                  └─────────────┘
```

### 时序说明

1. **STM32 → 云端**：定时上报传感器数据（如每 1s 一次）
2. **云端 → STM32**：小程序操作后下发控制指令
3. **STM32 响应**：执行动作后再次上报状态确认

## 六、代码模板

### STM32F103C8T6 参考代码

```c
#include "stm32f10x.h"
#include <stdio.h>

// 上报函数
void Report_Sensor_Data(float temp) {
    char buf[64];
    snprintf(buf, sizeof(buf), "{\"dp\":{\"shsihitemp\":%.2f}}\r\n", temp);
    USART_SendString(USART2, buf);
}

// 处理云端指令
void Process_Command(char *cmd) {
    if (strstr(cmd, "\"valve_control\":true") != NULL) {
        GPIO_SetBits(GPIOA, GPIO_Pin_0);  // 阀门开
    } else if (strstr(cmd, "\"valve_control\":false") != NULL) {
        GPIO_ResetBits(GPIOA, GPIO_Pin_0); // 阀门关
    }
}

int main(void) {
    SystemInit();
    USART2_Init();      // 连接蓝牙
    LED_Init();         // 调试用
    
    while (1) {
        float temp = Read_PT100_Temperature();  // 读取传感器
        
        Report_Sensor_Data(temp);                 // 上报云端
        
        // 等待并处理云端指令
        char rx_buf[128];
        if (USART2_GetData(rx_buf, sizeof(rx_buf))) {
            Process_Command(rx_buf);
        }
        
        delay_ms(1000);  // 每秒上报一次
    }
}
```

## 七、关键注意事项

| 问题 | 解决方案 |
|------|----------|
| 蓝牙模块功耗 | 休眠时降低上报频率 |
| JSON 字符串长度 | 控制在 128 字节以内 |
| 浮点数精度 | 使用 `%.2f` 或 `%.3f` 格式化 |
| 帧定界符 | 每条消息以 `\r\n` 结尾 |
| 并发问题 | 使用环形缓冲区处理串口数据 |
| 断线重连 | 实现看门狗自动重启蓝牙连接 |

## 九、调试技巧

1. **本地测试**：先用 USB 转 TTL 直连电脑，用串口助手查看 JSON 输出
2. **日志输出**：将关键步骤通过另一个串口打印到 PC
3. **小程序开发者工具**：先在不连硬件情况下模拟数据绑定
4. **抓包分析**：使用 Wireshark 或蓝牙调试 APP 查看实际传输内容
