蜂鸣器相关行为以 [Beep.md](Beep.md) 为准：无源蜂鸣器，TIM2_CH1 3kHz PWM 驱动，按键短鸣 100ms，报警页一直鸣叫，退出报警页后间歇鸣叫。
## 一、硬件连接

| 按键 | 丝印 | STM32 引脚 | 逻辑电平 |
|------|------|-----------|----------|
| KEY1 | K1   | PB12      | 低电平有效（Pull up）|
| KEY2 | K2   | PB13      | 低电平有效（Pull up）|
| KEY3 | K3   | PB14      | 低电平有效（Pull up）|
| KEY4 | K4   | PB15      | 低电平有效（Pull up）|

**检测机制**: EXTI 下降沿中断（15_10_IRQHandler）+ 10ms 周期轮询回退

## 二、按键检测机制

### 状态机架构

```
第 0 级（轮询回退）
  └─ EXTI 中断未触发但引脚= LOW → 启动消抖

第 1 级（消抖确认）
  └─ 等待 20ms 电平稳定 LOW → 确认有效按下

第 2 级（事件判定）
  ├─ < 1s 释放 → 短按事件 → 入队
  └─ ≥ 1s 按住 → 长按事件 → 入队
```

### 时序图

```
t=0       EXTI 中断（或轮询检测到 LOW）
t=20ms    消抖窗口结束，确认按键按下
t=20ms~   等待释放或长按超时
t<1020ms  释放 → SHORT 事件
t=1020ms  仍按住 → LONG 事件（可重复触发）
```

### 事件队列

- 环形缓冲区，8 槽位
- 编码格式: `(key_id << 4) | event_type`
  - KEY2 短按 = `(2 << 4) | 1` = 0x21
  - KEY1 长按 = `(3 << 4) | 2` = 0x32
- 队列满时静默丢弃新事件

## 三、代码架构

```
key.h / key.c
├── Key_Init()       → 清零所有状态
├── Key_ISR()        → EXTI 中断：记录按下时间戳 + 标记消抖
├── Key_Scan()       → 10ms 周期：消抖状态机 + 事件生成
└── Key_GetEvent()   → 主循环消费：从队列取事件
```

## 四、关键注意事项

| 要点 | 说明 |
|------|------|
| **引脚映射** | 必须用数组 `{PIN_12, PIN_13, PIN_14, PIN_15}` 而非 `PIN_12 + i`。因为 HAL GPIO_PIN_x 是位掩码 (`1<<x`)，PIN_12+1=0x1001 ≠ PIN_13=0x2000 |
| **消抖在轮询中** | EXTI 中断只记录时间戳，消抖和事件判定均在 `Key_Scan()` 中完成，减少 ISR 耗时 |
| **轮询回退** | 当系统中断负载高时，EXTI 可能延迟触发。第 0 级轮询回退确保按键不被遗漏 |
| **last_debounce_tick** | 必须放在 `Key_t` 结构体中（而非 static 局部变量），否则看门狗复位后值不丢失会导致状态机永久阻塞 |
| **调用周期** | `Key_Scan()` 必须每 10ms 调用一次，`Key_GetEvent()` 在消费循环中调用直到返回 0 |
| **长按行为** | 长按仅在本系统中部分保留，大部分页面仅使用短按。长按超 1s 后会重复触发 |


按键本身只是信号源，全部业务逻辑在 OLED 模块中解释。

## 五、蜂鸣器按键反馈（非阻塞设计）

### 原理

- `Buzzer_Trigger()` 只记录开始时刻 + 启动 PWM，立即返回
- `Buzzer_Process()` 在主循环中轮询，到时间自动关 PWM
- 全程不调用 `HAL_Delay`，不阻塞其他模块

### 状态机

```
IDLE → (触发) → BUZZING → (100ms 到) → IDLE
```

### 接口

```c
void Buzzer_Trigger(void);   // 按键事件调用，非阻塞
void Buzzer_Process(void);   // 主循环每轮调用
```

### 主循环调用关系

```c
while (1) {
    Key_Process();       // 出队按键 → 确认后调 Buzzer_Trigger()
    Sensor_Update();     // 读传感器
    OLED_UpdateDisplay();// 刷新显示
    Buzzer_Process();    // 检查计时，关 PWM
}
```

### 与报警音的共存（后期）

`Buzzer_Trigger()` 加优先级参数：报警 > 按键提示音。报警中忽略按键触发，报警结束后恢复。
