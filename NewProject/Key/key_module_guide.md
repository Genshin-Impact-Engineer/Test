# 独立按键模块指南（电子秤/POS 系统）

## 一、硬件连接

| 按键 | 丝印 | STM32 引脚 | 逻辑电平 |
|------|------|------------|----------|
| KEY0 | K1 | PB12 | 低电平有效（外部上拉）|
| KEY1 | K2 | PB13 | 低电平有效（外部上拉）|
| KEY2 | K3 | PB14 | 低电平有效（外部上拉）|
| KEY3 | K4 | PB15 | 低电平有效（外部上拉）|

**检测机制**: EXTI 下降沿中断（15_10_IRQHandler）+ 10ms 周期轮询回退

## 二、按键功能映射（三页面）

### Page_one：称重页

| 按键 | 未选中 | 选中类别 | 选中单价 | 编辑模式 |
|------|--------|----------|----------|----------|
| KEY0 | 光标↑ 0→1→2→0 | — | — | 退出编辑 |
| KEY1 | 光标↓ 0→2→1→0 | — | — | 下一商品 / 单价-0.5 |
| KEY2 | **结算**（→预览页）| 进入编辑 | 进入编辑 | 确认退出编辑 |
| KEY3 | **重称** | 退出选中 | 退出选中 | 退出编辑 |

### Page_two：预览编辑页

| 按键 | 未选中 | 选中类别（编辑）| 选中单价（编辑）|
|------|--------|-----------------|-----------------|
| KEY0 | 光标↑ | 退出选中 | 退出选中 |
| KEY1 | 光标↓ | **下一商品** | **单价 -0.5** |
| KEY2 | **保存修改，返回称重** | 保存并返回 | 保存并返回 |
| KEY3 | **放弃修改，返回称重** | 放弃并返回 | 放弃并返回 |

### Page_three：报警页

| 按键 | 功能 |
|------|------|
| 任意 KEY0/KEY1/KEY2/KEY3 | **退出报警**，返回称重页，停止蜂鸣器 |

## 三、按键检测机制

### 状态机架构

```
第 0 级（轮询回退）
  └─ EXTI 中断未触发但引脚= LOW → 启动消抖

第 1 级（消抖确认）
  └─ 等待 20ms 电平稳定 LOW → 确认有效按下

第 2 级（事件判定）
  ├─ < 1s 释放 → 短按事件 → 入队
  └─ ≥ 1s 按住 → 长按事件 → 入队
```

### 时序图

```
t=0       EXTI 中断（或轮询检测到 LOW）
t=20ms    消抖窗口结束，确认按键按下
t=20ms~   等待释放或长按超时
t<1020ms  释放 → SHORT 事件
t=1020ms  仍按住 → LONG 事件（可重复触发）
```

### 事件队列

- 环形缓冲区，8 槽位
- 编码格式: `(key_id << 4) | event_type`
  - KEY2 短按 = `(2 << 4) | 1` = 0x21
  - KEY3 长按 = `(3 << 4) | 2` = 0x32
- 队列满时静默丢弃新事件

## 四、代码架构

```
key.h / key.c
├── Key_Init()       → 清零所有状态
├── Key_ISR()        → EXTI 中断：记录按下时间戳 + 标记消抖
├── Key_Scan()       → 10ms 周期：消抖状态机 + 事件生成
└── Key_GetEvent()   → 主循环消费：从队列取事件
```

## 五、关键注意事项

| 要点 | 说明 |
|------|------|
| **引脚映射** | 必须用数组 `{PIN_12, PIN_13, PIN_14, PIN_15}` 而非 `PIN_12 + i`。因为 HAL GPIO_PIN_x 是位掩码 (`1<<x`)，PIN_12+1=0x1001 ≠ PIN_13=0x2000 |
| **消抖在轮询中** | EXTI 中断只记录时间戳，消抖和事件判定均在 `Key_Scan()` 中完成，减少 ISR 耗时 |
| **轮询回退** | 当系统中断负载高时，EXTI 可能延迟触发。第 0 级轮询回退确保按键不被遗漏 |
| **last_debounce_tick** | 必须放在 `Key_t` 结构体中（而非 static 局部变量），否则看门狗复位后值不丢失会导致状态机永久阻塞 |
| **调用周期** | `Key_Scan()` 必须每 10ms 调用一次，`Key_GetEvent()` 在消费循环中调用直到返回 0 |
| **长按行为** | 长按仅在本系统中部分保留，大部分页面仅使用短按。长按超 1s 后会重复触发 |

## 六、与新项目的适配

本按键模块的**硬件层代码完全不变**。适配新项目只需修改：

1. `OLED_HandleKey()` 中的按键行为解释（oled.c）
2. 长按页面导航逻辑（如需要重新启用）

按键本身只是信号源，全部业务逻辑在 OLED 模块中解释。
